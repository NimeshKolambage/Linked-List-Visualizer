What is a Linked List?

Think of a Linked List as a series of connected train cars. Instead of being stored in one big block of memory, each piece of data points to the next, allowing for a dynamic and flexible structure.

01

The Building Block: The Node

Every Linked List is made of Nodes. A Node is a simple container with two essential parts:

  • Data: The actual information you want to store (like a number or string).
  • Pointer: The memory address that tells the computer where the next "car" in the train is located.

Understanding the Three Variations

Linear

Singly Linked List

The simplest form. Each node points only to the next node. Navigation is strictly one-way—forward only.

Example: A simple "Undo" stack.
Bidirectional

Doubly Linked List

Features two pointers: Next and Previous. This allows you to traverse the list in both directions with ease.

Example: Browser "Back" and "Forward" buttons.
Looping

Circular Linked List

The last node’s pointer doesn't lead to null; it points back to the Head. This creates an endless loop.

Example: A Music Player's "Repeat All" feature.
